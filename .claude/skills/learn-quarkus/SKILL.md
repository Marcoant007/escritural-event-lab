Ensine o tema: $ARGUMENTS

Use esta estrutura:

1. Conceito em linguagem direta.
2. Onde ele aparece ou deveria aparecer neste repositório.
3. O que acontece no build time e no runtime do Quarkus.
4. Thread, transação e escopo CDI envolvidos, quando aplicável.
5. Erros comuns.
6. Diferença para Spring apenas quando ajudar a compreensão.
7. Um experimento pequeno para observar o comportamento.
8. Perguntas de revisão para eu responder.
9. Não faça uma grande implementação automaticamente; altere código somente se o argumento pedir explicitamente um experimento.
